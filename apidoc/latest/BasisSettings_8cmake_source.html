2 # @file BasisSettings.cmake
3 # @brief Default project-independent settings.
4 #
5 # This module defines global CMake constants and variables which are used
6 # by the BASIS CMake functions and macros. Hence, these values can be used
7 # to configure the behavior of these functions to some extent without the
8 # need to modify the functions themselves.
9 #
10 # @note As this file also sets the CMake policies to be used, it has to
11 # be included using the @c NO_POLICY_SCOPE in order for these policies
12 # to take effect also in the including file and its subdirectories.
13 #
14 # @attention Be careful when caching any of the variables. Usually, this
15 # file is included in the root CMake configuration file of the
16 # project which may also be a module of another project and hence
17 # may overwrite this project's settings.
18 #
19 # @attention Keep in mind that this file is included before any other
20 # BASIS module. Further, project-specific information such as
21 # the project name are not defined yet.

443 ## @brief Enable the installation of scripted modules in site specific default directories.
444 #
445 # If this option is @c ON, scripted modules such as Python and Perl modules, in particular,
446 # are installed in the default installation directories for site packages of the respective
447 # interpreter. This means that these modules may be installed outside the installation
448 # root directory as specified by the @c CMAKE_INSTALL_PREFIX. When this option is set to
449 # @c OFF, all modules are installed in subdirectories of the @c CMAKE_INSTALL_PREFIX.
450 # These directories may have to be added to the search path for modules manually as they
451 # might not be in the default search path of the respective interpreter.
452 #
453 # The installation directories for public modules which are intended for external use
454 # can further be set using the -D option of CMake or be modified by editing the respective
455 # advanced CMake cache variables named <tt>INSTALL_&lt;LANG%gt;_SITE_DIR</tt>.
456 #
457 # @note Even though it is more convenient for the user of a package to have the modules
458 # being installed in the default directories, this option is set to @c OFF by default.
459 # The reasons are that it is in first place expected that the installation will copy
460 # files only to directories within the @c CMAKE_INSTALL_PREFIX. Moreover, it is not
461 # guaranteed that the user has write permissions for the default site packages directories.
462 # Last but not least, when installing public modules located in the @c PROJECT_LIBRARY_DIR
463 # source directory, BASIS does not set a default module @c PREFIX which reduces the risk
464 # of overwriting existing modules of other packages. If the developer of a BASIS package
465 # was not thorough enough and did not follow the guidelines, setting this option to @c ON
466 # has the potential risk of overwriting other packages' modules. Therefore,
467 # modules are only installed in system default locations if explicitly requested.
468 option (BASIS_INSTALL_SITE_PACKAGES "Enable the installation of scripted modules in site specific default directories." OFF)
469 mark_as_advanced (BASIS_INSTALL_SITE_PACKAGES)
