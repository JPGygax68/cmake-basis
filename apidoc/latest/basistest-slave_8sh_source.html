38  cat - << EOF-OPTIONS
39 Optional arguments:
40  --project, -p The name of the project to be tested.
41  --branch, -b The branch to be tested, e.g., "tags/1.0.0".
42  Default: "trunk".
43  --model, -m The name of the dashboard model, i.e., either "Nightly",
44  "Continuous", or "Experimental".
45  Default: "Experimental".
46  --script, -S CTest script which performs the testing.
47  Default: basistest.ctest script of BASIS.
48  --args, -a Additional arguments for the CTest script. See below.
49  Instead of using the --args option, the additional arguments
50  can be given directly with only two dashes (--) prefixed.
51 
52 Standard arguments:
53  --verbose, -v Increases verbosity of output messages. Can be given multiple times.
54  --help, -h Print help and exit.
55  --helpshort Print short help and exit.
56  --version Print version information and exit.
57 
58 All other arguments are passed on as additional arguments to the CTest script
59 without the leading two dashes (--). For example, the optional argument
60 --coverage corresponds to supplying the option --args with the value 'coverage'.
61 Both result in the argument 'coverage' being passed on to the CTest script.
62 See below for a list of valid arguments of the CTest script.
63 EOF-OPTIONS
